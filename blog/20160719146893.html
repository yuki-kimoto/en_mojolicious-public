Mojolicious 7.0 released: Perl real-time web framework

Translation

I'm very excited to report the release of Mojolicious 7.0 (Doughnut).

While the Mojolicious community continues to grow, the size of the division is actually getting smaller. You will be surprised. This is a very good thing. Despite all the features we've added in the last few years, we've continued to reduce the code size from 11247 lines in 2010 to 8490 lines today.

As you can see, it's not a new trend, so we emphasize iterative improvement. It involves finding a more efficient solution to a problem that has already been resolved.

At the same time, the number of tests has increased in all releases, with 11245 tests reaching a very comfortable 95%test coverage score.

With every release, we can see significant performance improvements. This can be seen again in 7.0, even though finding optimization opportunities continues to be difficult.

You can see a particular trend in the daily benchmark of "requests per second".

In addition, we've done some internal refactoring. This is so that the built-in HTTP and WebSocket servers can scale efficiently, even with a large number of CPU cores. Large numbers of CPUs will become more commonly used.

The main focus of the year was stability, but new features have also been added. Here is a list of highlights.

  • Named variables in all templates : Mojo::Template is native and supports named variables
  • HTTP methods: "_method" query parameters can be used to override request methods
  • SNI: Available on the server side
  • Windows: Full support for Ubuntu on Windos. This includes zero downtime software upgrades
  • Validation Filters: Parameters can be filtered before validation
  • WebSocket subprotocol: Supports WebSocket subprotocol negotiation
  • Minion: Our small job queues are faster, supporting parallel processing, multiple named queues, automatic reruns of failed jobs, and job dependency.
  • Code Conduct: Diversity is bothering us, so we decided to apply Code Conduct.

More can be found by looking at Changes on GitHub for a complete list of improvements.
